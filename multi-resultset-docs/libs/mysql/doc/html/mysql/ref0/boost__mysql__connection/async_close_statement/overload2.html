connection::async_close_statement (2 of 2 overloads)

Closes a statement, deallocating it from the server.

Synopsis 
template<
    class CompletionToken>
auto
async_close_statement(
    const statement& stmt,
    diagnostics& diag,
    CompletionToken&& token);
Description

After this operation succeeds, this->valid() will return false, and no further functions may be called on this prepared statement, other than assignment.

This operation involves only writes on the underlying stream. The handler signature for this operation is void(boost::mysql::error_code).
