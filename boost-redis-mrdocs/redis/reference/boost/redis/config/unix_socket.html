boost::redis::config::unix_socket

The UNIX domain socket path where the server is listening. If non‐empty, communication with the server will happen using UNIX domain sockets, and addr will be ignored. UNIX domain sockets can't be used with SSL: if unix_socket is non‐empty, use_ssl must be false.
