This class is a handle to a server-side prepared statement. The main use of a prepared statement is executing it using prepared_statement::execute, which yields a resultset. Prepared statements are default-constructible and movable, but not copyable. prepared_statement::valid returns false for default-constructed and moved-from prepared statements. Calling any member function on an invalid prepared statements, other than assignment, results in undefined behavior. Prepared statements are managed by the server in a per-connection basis: once created, a prepared statement may be used as long as the parent connection object (i.e. the connection that originated the resultset) is alive and open. Calling any function on a prepared_statement whose parent connection has been closed or destroyed results in undefined behavior.
