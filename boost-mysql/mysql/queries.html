Text queries

This section describes issuing text SQL queries in detail.

Text queries are started by calling the connection::query and connection::async_query functions. They accept a SQL text query as parameter, which will be executed in the MySQL server. They return a resultset with the query results. This is commonly called the "text protocol" in MySQL slang.

Almost any query that may be issued in the mysql command line can be executed using this method. This includes SELECTs, UPDATEs, INSERTs, DELETEs, CREATE TABLEs... In particular, you may start transactions issuing a START TRANSACTION, commit them using COMMIT and rolling them back using ROLLBACK. You can also access the prepared statement functionality by issuing PREPARE and EXECUTE commands. However, it is advised to use the dedicated prepared statement functionality in Boost.Mysql for this task.

Note

You can only execute a single query at a time. Passing in several queries separated by semicolons won't work.

Query strings are assumed to be encoded using the connection's character set. You can set this value when establishing the connection (see this section) or change it using a SET NAMES statement.

Note

Client-side SQL query composition is not available. This is considered the resposibility of a higher-level component rather than Boost.Mysql's. Prepared statements offer server-side query composition that should cover most needs without the potentital security issues that client-side composition may introduce.

This example shows how to use sync query functions. There are also examples covering the use of async queries with callbacks, futures, Boost.Coroutine coroutines and C++20 coroutines.
