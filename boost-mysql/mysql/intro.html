Introduction

Boost.Mysql is a C++11 client for the MySQL database server, based on Boost.Asio. This library is in the process of being proposed for Boost.

Motivation

MySQL is one of the most widespread SQL database servers. MySQL clients connect to the server in order to issue SQL queries. For this purpose, MySQL employs a dedicated protocol. Boost.Mysql is an implementation of the client side of this protocol.

The design goals of this library are:

  • Ease of use: designed to be intuitive and hide the MySQL protocol complexities.
  • Interoperability with Asio: this library employs the same principles as Boost.Asio and Boost.Beast. Users of any of these libraries will immediately understand Boost.Mysql, and will have it easy to integrate it in their programs.
  • Basis for further abstraction: this library is focused on doing a single thing and doing it well. For this reason, it can be used easily by higher level components as a building block.
  • Efficiency.

Requirements

Boost.Mysql requires:

  • C++11 capable compiler (tested with gcc 7.4, clang 7.0, Apple clang 11.0, MSVC 19.25).
  • Boost. Boost.Mysql does not work with the standalone version of Boost.Asio.
  • OpenSSL.

Boost.Mysql is a header-only library. The following additional libraries are needed at link time:

  • OpenSSL is always required.
  • Boost.Coroutine is required when using coroutines created by boost::asio::spawn.

If you are using CMake (finding the library using find_package or fetching it using FetchContent), you can link against the Boost::mysql* CMake target, which will take care of the details for you.

Boost.Mysql has been tested with the following versions of MySQL:

  • MySQL v5.7.29.
  • MySQL v8.0.19.
  • MariaDB v10.3.
