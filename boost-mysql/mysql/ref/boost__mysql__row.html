Description

Call row::values to get the actual sequence of [reflinl value]s the row contains. There will be the same number of values and in the same order as fields in the SQL query that produced the row. You can get more information about these fields using resultset::fields. If any of the values is a string, it will point to an externally owned piece of memory. Thus, the row class is not owning, as opposed to owning_row. Note that, in any case, the sequence of values is owned by the row object. The distinction applies only to the memory pointed to by string values.
