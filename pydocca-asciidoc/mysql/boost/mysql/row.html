boost::mysql::row

Defined in header <boost/mysql/row.hpp>

An owning, read-only sequence of fields.

class row;

Although owning, row is read-only. It’s optimized for memory re-use. If you need to mutate fields, use a std::vector instead (see as_vector and as_vector).

Object lifetimes

A row object owns a chunk of memory in which it stores its elements. On element access (using iterators, at or operator[]) it returns field_view's pointing into the row’s internal storage. These views behave like references, and are valid as long as pointers, iterators and references into the `row remain valid.
